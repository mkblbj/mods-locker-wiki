## 🚀 2分で始める（TL;DR）

**ステップ1｜ディレクトリ設定**

1. **設定 → ディレクトリ設定**を開く
2. **ゲームディレクトリ**を選択（2K25/26のルートディレクトリを指定；`…/mods`を選択しないでください）
3. **Modsライブラリディレクトリ**を選択

![簡単な画像ホスティング - EasyImage](https://pic.x-yue.top/i/2025/10/10/zewzdg.png)

**ステップ2｜Modsインポート**

1. **Mods情報管理（またはModsライブラリ）**に入る
2. **スキャンして更新**をクリック（ライブラリディレクトリをスキャンし、新しいフォルダを発見したらワンクリックでインポート）
3. チェックを入れる → **選択したものをインポート**（インポートパネルがある場合）
4. カテゴリを設定 → modsを**カテゴリに追加**
5. インポートパネルは**プレビューと一括リネーム**をサポート（インポート後も情報ページで変更可能）

![簡単な画像ホスティング - EasyImage](https://pic.x-yue.top/i/2025/10/10/zewvwx.png)

**ステップ3｜有効化と自動展開（新機能）**

1. **Mods管理**を開き、スイッチを切り替えて**有効化/無効化**
2. システムは約**2秒**以内に自動的に**増分展開**を実行（上部に「展開待ち…」と進捗表示）
3. もう「有効化されたModsを展開」を手動でクリックする必要はありません
4. 完全にクリーンな状態に戻すには：右上の**クリーンアップしてリセット**をクリック（リンクをクリーンアップしてすべて無効化）

![簡単な画像ホスティング - EasyImage](https://pic.x-yue.top/i/2025/10/10/zewwds.png)

## 🔎 1分自己チェック（最もよくある問題）

- **ディレクトリが正しくない** → ゲームディレクトリは**2K25/26のルート**を選択し、`…/mods`ではありません  
- **有効化したばかりで反映されない** → 約**2秒**待って自動展開；上部に「展開待ち…」バッジと進捗表示；異常がある場合は**クリーンアップしてリセット**後に再試行  
- **階層が異常** → 各Modは**個別のフォルダ**（以下の例を参照）  
- **権限/セキュリティソフト** → リンクを作成できない場合、管理者として実行するかセキュリティソフトを許可  
- **無料版の有効化制限** → 無料版は最大**10個のMods**を有効化可能；上限に達すると「アップグレード」の提示

## 📁 ファイルレベル管理（新機能）

1. **Mods情報管理**を開く → 特定のModを選択 → **ファイル構造を表示**をクリック / またはModsカードをダブルクリック
2. ファイルツリーで個別の**ファイル/フォルダ**の有効化/無効化を制御
3. すべての変更は**自動増分展開**キューに入る（上部に「展開待ち…」表示）
4. デフォルトに戻すには：**ファイル制御をワンクリックリセット**をクリック

> 推奨：特定のファイルを置換/除外する必要がある場合のみファイルレベル制御を使用；通常はデフォルトのままで問題ありません。

## 🧩 プラグインストアと試用

1. メインページの**ストア**を開き、利用可能なプラグインを閲覧してワンクリックで有効化/試用
2. 新規インストール時にデフォルトで**3日間の試用を自動開始**；ストアで試用コード`TRIAL-ALLADDONS-...`を入力することも可能
3. 試用/有効化期間中、プラグイン機能（IFF Fixerなど）はすぐに利用可能、**再起動不要**
4. 試用期間終了後、関連機能が制限され、ストアまたはライセンスページでアップグレード可能

![簡単な画像ホスティング - EasyImage](https://pic.x-yue.top/i/2025/10/10/j9k04.png)

## 🧰 IFF Fixer修復ツール（2K26 1.4）

- 適用：NBA 2K26を1.4にアップグレード後、一部のModsでIFF互換性問題が発生した場合
- 入口：**Mods情報管理**ページで**IFFファイルを修復**をクリック（または対応する入口）
- 機能：**ワンクリック修復**、**一括処理**、**スマート検出**（必要なファイルのみ修復）
- 推奨：操作前にModsディレクトリをバックアップ；修復完了後、ファイルレベル管理で変更を確認

## 🗂️ 推奨ディレクトリと命名（必須、管理しやすい）

::: details 推奨構造（各Modは個別フォルダ）

**推奨される2つの方法：**

- `あなたのModsライブラリディレクトリ/<mod-name>/mods/<具体的な内容>/`  
  例：`face-kobe08/mods/char/sig/png...`
- `あなたのModsライブラリディレクトリ/<mod-name>/<具体的な内容>/`  
  例：`face-kobe08/char/sig/png...`

**命名例：**

- コービーフェイス：`face-kobe08/mods/char/sig/png...` または `face-kobe08/char/sig/png...`
- レイカーズ2010ファイナルジャージ：`jersey-01lakers-finals/mods/char/...` または `jersey-01lakers-finals/char/...`
- コービー10クリスマスシューズ：`shoes-kobe10-christmas/clothing/...` または `shoes-kobe10-christmas/mods/clothing/...`
- 例えばバイナムフェイス

**ヒント：**

- できるだけ**英語で命名**し、統一されたスタイルで簡潔明瞭に

## ❓FAQ

**Q1：インポート失敗した場合はどうすればいいですか？**  
ディレクトリが正しく設定されているか、ディレクトリ階層が規範に沿っているか確認してください（「推奨構造」を参照）。

**Q2：Modiumが起動しない/リストが応答しない？**  
**Library / NBA2K25(26)**ページにいることを確認；Modiumパスと権限を確認。

**Q3：バックアップと復元方法は？**  
**Modsライブラリディレクトリ**をバックアップするだけ；展開はリンクなので、クリーンアップで復元できます。

**Q4：切り替えた後に展開が表示されないのはなぜですか？**  
有効化後、システムは約**2秒**以内に**増分方式**で自動展開；期間中、上部に「展開待ち…」と進捗が表示されます。まだ異常がある場合：1) 設定のパスを確認；2) **クリーンアップしてリセット**をクリック後に再試行。

**Q5：ファイルレベルの有効化/無効化と全体スイッチの違いは？**  
全体スイッチはMod全体の有効化/無効化を制御；ファイルレベルは特定のファイルやサブフォルダを細かく除外/含めるために使用し、競合や複数バージョンの共存シナリオに適しています。

**Q6：試用の開始方法/期限切れ時の対応は？**  
初回インストール時に**3日間の試用**を自動開始を試みます；「ストア」で試用コード`TRIAL-ALLADDONS-...`を入力することも可能。期限切れ後、関連するプラグイン機能が制限され、ストアまたはライセンスページでアップグレードできます。
